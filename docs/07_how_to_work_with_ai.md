# 怎麼與 AI 解決真實世界的問題

## 務必使用 Colab 或類似的服務

> 不是因為 Colab 有多好用，而是擔心你把原始檔案弄壞了。

首先，不管怎麼樣都「**把你打算用到的檔案、資料夾、或任何東西上傳到 Google Colab**」，如果資料很多，先上傳一部份就好。  
  
這樣做的目的是防止你把自己的電腦或資料弄壞。 即使是我自己，三不五時還是會不小心刪掉一些重要的檔案或資料夾，或是把檔案編輯成沒有人認得出來的樣子。 但我知道怎麼恢復它們（*其實也常常恢復不了* 😢），所以我不是太擔心這個問題。  
  
但新手請千萬千萬不要這樣做，不然你很可能連哪裡出了錯都不知道。 利用 Google Colab 進行資料處理是一個很好的保護，就算不小心把資料弄壞了，大不了全部刪掉再上傳一次就好了。  

> 如果是包含個資或機密資訊的檔案，建議去除敏感資料後再上傳。

## 有兩種跟 AI 協作的方式

### 自己手動拆積木
  
試著把你要做的事情利用前面教過的概念，如：「變數、`if`, `for`」等，拆解成小步驟。  
  
舉例來說，你想『把某個資料夾中所有 `csv` 檔案裡的「地區」欄位等於「台北、嘉義、或台南」的資料都找出來，然後儲存為一個新的 `csv` 檔案，檔名是 `真的是最終版資料了.csv`。』  
  
我們可以這樣拆解成小積木：  

1. 先把 `XX資料夾下` 所有 `csv` 檔案都讀進來，然後把它們放到一個變數裡
2. 接著，把「地區」欄位等於「台北、嘉義、或台南」的資料都找出來  
3. 最後，把這些資料儲存為一個新的 `csv` 檔案，檔名是 `真的是最終版資料了.csv`  

> 第一個步驟用到 `for` 的概念，第二個步驟用到 `if` 的概念。

---

接著，我們「一步一步」請 AI 幫我們把這些小步驟變成程式碼。  
首先，讓 ChatGPT 清楚知道你要做什麼、避免通靈：

```
你是一名資深的 Python 工程師，我正在 Google Colab 進行程式開發。
請協助我生成適合的 Python 代碼，並附上清楚易懂的繁體中文註釋。
如果對我的指令有不清楚的地方，請先問我、釐清後再進行生成。
```

接著再逐段輸入需求，並要求 AI 列出必要的資訊讓你檢查是否每一步都如你所想。

```
先把 `XX資料夾下` 所有 `csv` 檔案都讀進來、把它們放到一個變數裡。  
並印出相關資訊，讓我確認是否正確讀取。
```

後續每一段都像上面一樣，逐步執行、逐步檢查。  
這樣能確保每個步驟都是你想要的結果。只要某個步驟發現有問題，就能馬上修正。  

> 分段測試可以讓你在最早的時候發現問題，出錯時也容易知道是哪一步出問題，大幅減少 debug 的時間和困擾。

---

### 委託 AI 拆解需求

另一種做法是放棄自己「拆積木」的過程，交給 AI 來執行。  
只要把你的需求清楚告訴 AI，請它自動幫你拆解、寫出可直接執行的程式碼。  

```
你是一名資深的 Python 工程師。
我正在 Google Colab 開發程式，請幫我把下面的需求拆解成多個簡單、可驗證的步驟，  
每個步驟都附上可以直接執行的 Python 代碼，並加上清楚的繁體中文註解，讓我逐步檢查和測試。

《需求》
請把 `XX` 資料夾裡所有 `csv` 檔案中，「地區」欄位等於「台北、嘉義、台南」的資料都找出來，
然後存成一個新的 `csv` 檔案，檔名請用 `真的是最終版資料了.csv`。

```

使用步驟：  

1. 把需求貼給 AI，請它幫你拆解。  
2. 仔細檢查每一段說明和程式碼，發現有問題直接回饋請它只修正那一段。  
3. 一定要分段執行，每一步都讓 AI 印出資訊，確認結果正確才繼續下一步。  
  
這兩種協作方式沒有絕對好壞，選擇你覺得最舒服的方式就好。  
唯一要記得的是：「分段執行，錯誤容易排查，開發效率更好！」

> 遇到不懂的地方，可以直接問AI：『你剛剛寫的這一段在做什麼？ 請用白話解釋。』

## 程式出錯直接問 AI 就對了

> 不管新手還是老手都會寫錯，  
把「你的程式碼」和「完整錯誤訊息」一起貼給 AI，  
它通常 1 分鐘內就能幫你找到原因和解法，比你自己亂猜快多了。

我建議你可以先用下面的 prompt 詢問，未來再慢慢修改成你的風格：

```
你是一名資深 Python 除錯工程師。  
我在 Colab 撰寫 Python 時出現錯誤，請協助我下面幾件事：  
  1. 告訴我為什麼會出現這個錯誤？  
  2. 你建議怎麼修復？  
  3. 如果需要修改程式碼，請在 markdown block 中提供我修改後的完整程式碼，並附上清楚易懂的繁體中文註釋。  

《這是我的程式碼》
（把你的程式碼貼在這裡）

《這是完整的錯誤訊息》
（把錯誤訊息貼在這裡，從第一行到最後一行，不要只貼中間幾行）
```

> AI 寫在 markdown block 中的程式碼，可以讓你直接一鍵複製，格式還不會跑掉。

## 怎麼下好 prompt，讓 AI 聽懂你要什麼

AI 越來越聰明，但它還是不會讀心。  
你指令給得越明確，得到的回應就越符合需求、省下更多溝通時間。  
  
只要抓住這五個關鍵元素，你下的 prompt 通常不會差：

1. **角色**：讓 AI 知道它該扮演誰？
2. **上下文**：你的背景、工作環境是什麼？
3. **任務**：你要 AI 幫你做什麼？  
4. **條件/限制**：有什麼規則、注意事項？  
5. **輸出格式**：你要 AI 用什麼格式回答你？

這些要素不一定要按照順序出現，但當你在處理複雜問題時，最好能夠兼顧這幾個面向。

---

**不好的例子**

```
幫我合併兩個 Excel 檔案。
```

**好的例子**
```
你是一名資深 Python 工程師。
我在 Google Colab 有兩個 Excel 檔案：sales_Q1.xlsx 和 sales_Q2.xlsx，兩者結構相同。
請幫我寫一段 Python 程式，把這兩個檔案依照欄位名稱合併成一個 DataFrame，然後存成新的 Excel（檔名請用 sales_1H.xlsx）。
請加上繁體中文註解，並印出合併後的資料列數和欄位名稱以及前 5 筆資料。
```

## 你可能不需要知道這些東西

### 進階技巧：複雜錯誤怎麼問 AI

當你遇到更複雜、一次可能有多種修法的錯誤，可以用「二段式問法」請 AI 拆解選項。

**第一步：讓 AI 分析錯誤，給你多種修正方案**

```
你是一名資深 Python 除錯工程師。  
我在 Colab 撰寫 Python 時出現錯誤，請協助我下面幾件事：  
  1. 告訴我為什麼會出現這個錯誤？
  2. 你建議怎麼修復？
  3. 如果有多個修復建議，請分別列出來，並說明每個修復的優缺點。

《這是我的程式碼》
（貼上可以重現錯誤的程式碼）

《這是完整的錯誤訊息》
（從第一行到最後一行都要貼進來）
```

**第二步：根據選擇的修法，請 AI 寫出完整程式碼**

```
請按照《方案N》的方式進行修改，  
並在 markdown block 中提供我修改後的完整程式碼，附上清楚易懂的繁體中文註釋。
```

> 小提醒：如果不知道該選哪個，直接問 AI「你最推薦哪個方案？為什麼？」即可。
